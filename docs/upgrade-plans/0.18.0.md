## 1. Critical Changes (must apply before upgrade)

### 1.1 Observability — OpenTelemetry (Spring Boot 4 migration)

OpenTelemetry was migrated to Spring Boot 4 `spring-boot-starter-opentelemetry`. OTLP export is **disabled by default** (`OTEL_EXPORT_ENABLED=false`, replacing `OTEL_SDK_DISABLED=true`).

> [!IMPORTANT]
> If you currently export telemetry with `OTEL_SDK_DISABLED=false`, migrate the variables below **before** upgrading or OTLP export will silently stop. If you use collector auth headers via `OTEL_EXPORTER_OTLP_HEADERS`, that variable is **removed** — configure Spring header properties instead.

#### Migrate deprecated and removed variables


| Old variable                                | Replacement                                               | Notes                                |
| ------------------------------------------- | --------------------------------------------------------- | ------------------------------------ |
| `OTEL_SDK_DISABLED=false`                   | `OTEL_EXPORT_ENABLED=true`                                | Inverted semantics                   |
| `OTEL_SDK_DISABLED=true`                    | *(remove or omit)*                                        | Default behaviour — export stays off |
| `OTEL_LOGS_EXPORTER=otlp`                   | `OTEL_LOGS_EXPORT_ENABLED=true`                           | Per-signal boolean flags             |
| `OTEL_TRACES_EXPORTER=otlp`                 | `OTEL_TRACES_EXPORT_ENABLED=true`                         | Per-signal boolean flags             |
| `OTEL_METRICS_EXPORTER=otlp`                | `OTEL_METRICS_EXPORT_ENABLED=true`                        | Per-signal boolean flags             |
| `OTEL_EXPORTER_OTLP_PROTOCOL=http/protobuf` | `OTEL_EXPORTER_OTLP_TRANSPORT=http`                       | Value renamed                        |
| `OTEL_EXPORTER_OTLP_PROTOCOL=grpc`          | `OTEL_EXPORTER_OTLP_TRANSPORT=grpc`                       | Value unchanged                      |
| `OTEL_EXPORTER_OTLP_HEADERS`                | Spring `management.opentelemetry.*.export.otlp.headers.`* | **Removed** — no auto-mapping        |

## 2. High-Priority Changes

### 2.1 Security & RBAC — [Preview] Deployment Metrics Scrape

**[Preview] Deployment Metrics Scrape** — on-demand live metrics snapshot for any deployment type via `GET /api/v1/deployments/{id}/metrics`. Resource metrics (replica counts and per-pod CPU/memory) are available for all deployment types. Engine serving and operational metrics are scraped only for **INFERENCE** deployments; NIM and other types return those blocks as unavailable.

> [!NOTE]
> This is a **preview** feature. API and configuration may change in future releases.

Requires read-only RBAC:

- `pods/proxy` — INFERENCE engine metrics only (KServe namespace)
- `metrics.k8s.io` pods — when resource-usage block is enabled (Knative, NIM, and KServe namespaces)

Without these permissions, metrics requests fail at runtime — the application starts normally.

Add to the **KServe** namespace Role (INFERENCE engine metrics):

```yaml
  # [Preview] Deployment metrics scrape — INFERENCE engine /metrics via pod proxy
  - apiGroups: [""]
    resources: ["pods/proxy"]
    verbs: ["get"]
```

Add to **mcp-apps, kserve-models, and nim-models** namespace Roles (resource usage block):

```yaml
  # [Preview] Deployment metrics scrape — CPU/memory from metrics-server
  - apiGroups: ["metrics.k8s.io"]
    resources: ["pods"]
    verbs: ["get", "list"]
```

Configuration (enabled by default — set `METRICS_SCRAPE_ENABLED=false` to disable):


| Setting                                 | Default | Required (Yes/No) | Description                                                                           |
| --------------------------------------- | ------- | ----------------- | ------------------------------------------------------------------------------------- |
| `METRICS_SCRAPE_ENABLED`                | `true`  | No                | Master switch; when `false`, requests return HTTP 400.                                |
| `METRICS_SCRAPE_TIMEOUT_MS`             | `10000` | No                | Per-scrape timeout (ms) for INFERENCE pod `/metrics` via API-server proxy.            |
| `METRICS_SCRAPE_CACHE_TTL_MS`           | `5000`  | No                | Response cache TTL (ms) per deployment id.                                            |
| `METRICS_SCRAPE_RESOURCE_USAGE_ENABLED` | `true`  | No                | CPU/memory block from `metrics.k8s.io`; degrades gracefully if metrics-server absent. |

## 2. High-Priority Changes

### 2.1 Security & RBAC


| #   | Change                                                  | Action           | Details                                                                                                                                                                                                                                                                                                   |
| --- | ------------------------------------------------------- | ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 2   | `SECURITY_ROLES_MAPPING` + `READ_ONLY_ADMIN` role added | **Review / Add** | New role-based authorization model. Default mapping is `{}`. Without explicit mapping, `RolesMappingResolver` falls back to `allowedRoles` → all mapped to `FULL_ADMIN` (backward compat). Review if you want `READ_ONLY_ADMIN` users. Config import and MCP tool calling are restricted to `FULL_ADMIN`. |


**Helm values.yaml** ([dial-admin chart](https://github.com/epam/ai-dial-helm/tree/main/charts/dial-admin)) — configure role mapping (if desired):

```yaml
# dial-admin chart values.yaml
backend:
  env:
    SECURITY_ROLES_MAPPING: '{"ConfigAdmin":["FULL_ADMIN"],"Viewer":["READ_ONLY_ADMIN"]}'

manager:
  env:
    SECURITY_ROLES_MAPPING: '{"ConfigAdmin":["FULL_ADMIN"],"Viewer":["READ_ONLY_ADMIN"]}'
```

> Both components independently enforce role-based authorization. Set in `backend.env` for the admin UI and in `manager.env` for the deployment manager API. Use `backend.secrets` / `manager.secrets` if you prefer the values stored in a chart-managed Kubernetes Secret.

**Precedence** (from `RolesMappingResolver`):

1. `providers.*.roles-mapping` set → merged with default (provider wins on overlap)
2. `providers.*.allowed-roles` set → union with default, all mapped to `FULL_ADMIN`
3. `config.rest.security.default.roles-mapping` set → used as-is
4. `config.rest.security.default.allowedRoles` set → all mapped to `FULL_ADMIN`
5. Nothing configured → empty mapping, all requests 403

## 4. Low-Priority / Optional Changes

### 4.4 Networking & Ingress

- `HUGGINGFACE_DEFAULT_ALLOWED_DOMAINS` — default: `huggingface.co,transfer.xethub.hf.co,cas-server.xethub.hf.co`. Auto-added to Cilium egress for HuggingFace deployments.
- NIM Service Ingress Configuration — configurable Ingress template for externally exposed NIM services. Applied only when `K8S_NIM_DEPLOYMENT_USE_CLUSTER_INTERNAL_URL=false`. Generated Ingress host is `<NimServiceName>.<ClusterHost>`.


| Property                                                      | Default Value | Description                                        |
| ------------------------------------------------------------- | ------------- | -------------------------------------------------- |
| `app.nim-service-expose-ingress-config.enabled`               | `true`        | Enable Ingress for externally exposed NIM services |
| `app.nim-service-expose-ingress-config.annotations`           | *(see below)* | Annotations applied to the NIM Ingress resource    |
| `app.nim-service-expose-ingress-config.spec.ingressClassName` | `nginx`       | Ingress class name                                 |


Default annotations applied to NIM Ingress resources:


| Annotation                                       | Default Value              | Purpose                                                         |
| ------------------------------------------------ | -------------------------- | --------------------------------------------------------------- |
| `nginx.ingress.kubernetes.io/proxy-body-size`    | `"0"`                      | Unlimited request body size (needed for large prompts/images)   |
| `nginx.ingress.kubernetes.io/proxy-read-timeout` | `"600"`                    | 10-minute read timeout (LLMs can take time to stream responses) |
| `cert-manager.io/cluster-issuer`                 | `"letsencrypt-production"` | Cert-manager cluster issuer for automatic TLS certificates      |
| `nginx.ingress.kubernetes.io/force-ssl-redirect` | `"true"`                   | Force HTTPS redirect                                            |


TLS and routing rules are auto-generated from `app.nim.deploy.cluster-host` and the NIM service name. Each NIM service gets a TLS secret named `<NimServiceName>-tls-secret`.

> These properties are application-level config (not env vars). Override via `application.yaml` if needed. Defaults are safe for most nginx + cert-manager setups.

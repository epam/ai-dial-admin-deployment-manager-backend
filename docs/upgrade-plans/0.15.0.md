| 2   | `SECURITY_ROLES_MAPPING` + `READ_ONLY_ADMIN` role added | **Review / Add** | New role-based authorization model. Default mapping is `{}`. Without explicit mapping, `RolesMappingResolver` falls back to `allowedRoles` ‚Üí all mapped to `FULL_ADMIN` (backward compat). Review if you want `READ_ONLY_ADMIN` users. Config import and MCP tool calling are restricted to `FULL_ADMIN`. |


**Helm values.yaml** ‚Äî configure role mapping (if desired):

```yaml

  SECURITY_ROLES_MAPPING: '{"ConfigAdmin":["FULL_ADMIN"],"Viewer":["READ_ONLY_ADMIN"]}'
```

**Precedence** (from `RolesMappingResolver`):

1. `providers.*.roles-mapping` set ‚Üí merged with default (provider wins on overlap)
2. `providers.*.allowed-roles` set ‚Üí union with default, all mapped to `FULL_ADMIN`
3. `config.rest.security.default.roles-mapping` set ‚Üí used as-is
4. `config.rest.security.default.allowedRoles` set ‚Üí all mapped to `FULL_ADMIN`
5. Nothing configured ‚Üí empty mapping, all requests 403

## 4. Low-Priority / Optional Changes

### 4.1 Security & RBAC

- `SECURITY_REQUIRE_EMAIL` ‚Äî default: `false`. Set to `true` only if email must be present in JWT. Causes 401 if claim missing.

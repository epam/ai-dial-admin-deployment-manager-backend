# Data Model: Hubble Relay Domain Streaming

## New Entities

### `ImageBuildDomainEntryEntity`

**Table**: `image_build_domain_entries`
**Purpose**: Records each unique external domain accessed by an image build pod, with the Cilium verdict.

| Column | Java Type | DB Type (H2/PG/MSSQL) | Constraints |
|--------|-----------|----------------------|-------------|
| `id` | `Long` | `BIGINT` auto-increment | PK |
| `imageDefinitionId` | `UUID` | `UUID` / `UUID` / `UNIQUEIDENTIFIER` | NOT NULL, FK → `image_definition(id)` ON DELETE CASCADE. No separate `version` column — `imageDefinitionId` UUID is inherently version-specific (each build run gets a unique UUID row in `image_definition`). |
| `domain` | `String` | `VARCHAR(255)` | NOT NULL |
| `verdict` | `CiliumVerdict` (enum) | `VARCHAR(10)` | NOT NULL |
| `observedAt` | `long` | `BIGINT` (epoch ms) | NOT NULL |

**Unique constraint**: `(image_definition_id, domain, verdict)` — enforces deduplication per build run; `INSERT ... ON CONFLICT DO NOTHING` / `INSERT IGNORE` strategy at application level.

**Index**: `(image_definition_id)` for fast lookup by build; clustered on PK (`id`) for insertion-order replay.

---

### `DeploymentDomainEntryEntity`

**Table**: `deployment_domain_entries`
**Purpose**: Records each unique external domain accessed by a deployment pod for the current activation. Cleared on each new `POST /api/v1/deployments/{id}/deploy`.

| Column | Java Type | DB Type (H2/PG/MSSQL) | Constraints |
|--------|-----------|----------------------|-------------|
| `id` | `Long` | `BIGINT` auto-increment | PK |
| `deploymentId` | `String` | `VARCHAR(36)` | NOT NULL, FK → `deployment(id)` ON DELETE CASCADE |
| `domain` | `String` | `VARCHAR(255)` | NOT NULL |
| `verdict` | `CiliumVerdict` (enum) | `VARCHAR(10)` | NOT NULL |
| `observedAt` | `long` | `BIGINT` (epoch ms) | NOT NULL |

**Unique constraint**: `(deployment_id, domain, verdict)` — deduplication per deployment lifetime.

**Index**: `(deployment_id)` for fast lookup by deployment.

---

### `CiliumVerdict` enum

```java
public enum CiliumVerdict {
    ALLOWED,   // maps from Hubble flow verdict FORWARDED
    BLOCKED    // maps from Hubble flow verdict DROPPED
}
```

## Entity Relationships

```
image_definition (id: UUID)
    └── image_build_domain_entries (image_definition_id) [0..*] — ON DELETE CASCADE

deployment (id: VARCHAR(36))
    └── deployment_domain_entries (deployment_id) [0..*] — ON DELETE CASCADE
```

Entries in `deployment_domain_entries` are also explicitly deleted (DELETE WHERE deployment_id = ?) each time `POST /api/v1/deployments/{id}/deploy` activates a new deployment run, clearing the prior activation's domain history.

## JPA Repository Queries

### `ImageBuildDomainEntryJpaRepository`

```java
List<ImageBuildDomainEntryEntity> findByImageDefinitionIdOrderById(UUID imageDefinitionId);
```

Used by SSE domain polling (returns new entries above the last emitted `id` offset) and by build details endpoint (full domain list).

### `DeploymentDomainEntryJpaRepository`

```java
List<DeploymentDomainEntryEntity> findByDeploymentIdOrderById(String deploymentId);
```

Used by SSE domain polling for deployment pod log stream (replay + live) and future deployment details endpoint.
